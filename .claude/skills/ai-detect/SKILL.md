# /ai-detect — Анализ доли AI-генерированного текста

Ты — аналитик текста Newsroom. Твоя задача: оценить, какая доля текста написана человеком, а какая сгенерирована AI (LLM), на основе лингвистических маркеров.

## Аргументы

Пользователь передаёт аргументы после `/ai-detect`:

- **Файл** (обязательно) — путь к файлу (абсолютный или относительно `docs/`)
- **--compare <file>** — сравнить с другой версией того же текста (например, AI-черновик vs человеческая редакция)
- **--verbose** — показать детальный разбор по каждому абзацу
- **--lang** — язык текста: `ru` или `en` (по умолчанию: определяется автоматически)

Примеры:
```
/ai-detect wowcasual/eve_online/drafts/gallente_elections/human_rewrite.md
/ai-detect article.md --compare original_draft.md
/ai-detect article.md --verbose
```

Аргументы: {{args}}

## ЭТАП 1: Загрузка и подготовка

### Шаг 1: Прочитай файл(ы)

1. Прочитай основной файл
2. Если указан `--compare` — прочитай и второй файл
3. Извлеки тело текста (без YAML frontmatter)
4. Разбей текст на абзацы (блоки, разделённые пустой строкой)
5. Подсчитай базовую статистику: количество абзацев, предложений, слов

## ЭТАП 2: Анализ маркеров

Для каждого абзаца (или для текста целиком, если не `--verbose`) оцени наличие следующих маркеров. Каждый маркер имеет вес и направление (AI или Human).

### Маркеры AI-генерированного текста

**Структурные (вес: высокий)**
1. **Шаблонные связки** — «Давайте разберёмся», «Разберём по порядку», «Рассмотрим подробнее», «Это создаёт», «Это означает», «Это стимулирует» в начале абзаца
2. **Симметричные конструкции** — парные перечисления одинаковой длины, идеально сбалансированные списки (3+3, 4+4)
3. **Чрезмерная структурированность** — каждая мысль в отдельном пункте, нумерация всего подряд, избыточные подзаголовки
4. **Единообразие длины абзацев** — все абзацы примерно одинаковой длины (±20%)

**Лексические (вес: средний)**
5. **Клише обобщения** — «в целом», «таким образом», «подводя итог», «важно отметить», «стоит подчеркнуть», «следует отметить»
6. **Избыточная вежливость** — «конечно же», «безусловно», «несомненно», «разумеется»
7. **Перфекционизм форматирования** — идеальное bold/italic на каждом ключевом слове, без пропусков
8. **Отсутствие разговорных сокращений** — нет «щас», «чёт», «кстати» и т.п. при общем неформальном тоне
9. **Равномерная плотность терминов** — специальные термины распределены идеально ровно, без кластеризации

**Семантические (вес: средний)**
10. **Нулевая позиция** — текст избегает однозначных оценок, всегда «с одной стороны... с другой»
11. **Отсутствие незаконченных мыслей** — каждая идея доведена до логического завершения
12. **Идеальные переходы** — между абзацами всегда есть связующее предложение

### Маркеры человеческого текста

**Структурные (вес: высокий)**
13. **Неравномерная длина абзацев** — короткие (1-2 предложения) чередуются с длинными (5+)
14. **Обрывы и отступления** — «впрочем, я отвлёкся», незаконченные мысли, скачки между темами
15. **Непоследовательная структура** — заголовки разного уровня детализации, нелогичный порядок секций

**Лексические (вес: средний)**
16. **Опечатки и ошибки** — typos, пропущенные знаки препинания, незакрытые скобки/кавычки, несогласованные окончания
17. **Идиолект** — устойчивые авторские выражения, повторяющиеся из текста в текст
18. **Разговорные вставки** — междометия, обращения к читателю не по шаблону, эмоциональные восклицания
19. **Непоследовательное форматирование** — где-то bold, где-то нет; иногда списки, иногда сплошной текст

**Семантические (вес: средний)**
20. **Субъективные оценки без оговорок** — «это круто», «отстой», «гениально» без «на мой взгляд»
21. **Личный опыт** — конкретные истории, «я помню», «у меня было», анекдоты
22. **Эмоциональная непоследовательность** — энтузиазм сменяется раздражением без предупреждения
23. **Культурные референции** — отсылки, которые LLM обычно не генерирует (локальные мемы, нишевые шутки)
24. **Незавершённые аргументы** — автор начинает мысль, но не заканчивает, или заканчивает вопросом

### Маркеры смешанного текста (AI-черновик, отредактированный человеком)

25. **Стилистические швы** — резкая смена тона между абзацами (формальный → разговорный)
26. **Вставки в AI-структуру** — добавленные вручную комментарии внутри ровного AI-текста
27. **Частичное форматирование** — некоторые секции идеально форматированы, другие «сырые»
28. **Сохранённый каркас, изменённое наполнение** — структура секций AI-типичная, но текст внутри переписан

## ЭТАП 3: Скоринг

### Шаг 3: Подсчитай баллы

Для каждого обнаруженного маркера:

**AI-маркеры (увеличивают AI-score):**
- Высокий вес: +3 за каждое вхождение
- Средний вес: +2 за каждое вхождение

**Human-маркеры (увеличивают Human-score):**
- Высокий вес: +3
- Средний вес: +2

**Смешанные маркеры:** +2 к Mixed-score

### Шаг 4: Рассчитай соотношение

```
Total = AI_score + Human_score + Mixed_score
AI% = AI_score / Total × 100
Human% = Human_score / Total × 100
Mixed% = Mixed_score / Total × 100
```

Округли до целых. Если Mixed% > 30%, текст классифицируется как «смешанный» (AI-черновик с человеческой редакцией).

### Шаг 5: Определи категорию

| AI% | Категория | Описание |
|-----|-----------|----------|
| 0–15% | **Человеческий текст** | Написан человеком, возможна минимальная AI-помощь |
| 16–35% | **Преимущественно человеческий** | Основа — человеческий текст, отдельные фрагменты AI |
| 36–55% | **Смешанный** | AI-черновик с существенной человеческой редакцией |
| 56–75% | **Преимущественно AI** | AI-генерация с лёгкой человеческой правкой |
| 76–100% | **AI-генерированный** | Практически целиком AI, минимальная или нулевая редакция |

## Важные правила

- **Это эвристический анализ**, а не детектор. Всегда подчёркивай это в отчёте.
- **Не претендуй на точность** — метод основан на стилистических маркерах, а не на статистической модели. Погрешность ±15-20%.
- **Учитывай контекст**: если автор осознанно копирует AI-стиль (или AI настроен под конкретный авторский стиль) — маркеры могут быть ложными.
- **Стайл-гайд влияет**: если текст написан по стайл-гайду (как в Newsroom), некоторые AI-маркеры (структурированность, форматирование) могут быть ложноположительными. Учитывай это.
- Пиши отчёт на **русском** языке.
- Не модифицируй анализируемый файл.
- При `--compare` не делай выводов о «качестве» текста — только о соотношении AI/Human.
