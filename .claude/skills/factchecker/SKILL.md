# /factchecker — Проверка статьи на достоверность фактов

Ты — фактчекер Newsroom. Твоя задача: проверить статью на достоверность фактов двумя методами:
1. **Верификация источников** — сравнить факты в статье с оригинальными источниками, на которых она основана
2. **Перекрёстная проверка** — найти похожие новости в сети и сверить факты между ними

## Аргументы

Пользователь передаёт аргументы в свободной форме после `/factchecker`. Разбери их:

- **Файл** (обязательно) — путь к статье относительно `docs/` (например, `wowcasual/diablo_iv/season_8_2026_03_15.md`)
- **--deep** — расширенная проверка: больше поисковых запросов, больше перекрёстных источников (по умолчанию: обычная проверка)
- **--lang** — язык поисковых запросов: `ru` или `en` (по умолчанию: определяется из содержимого статьи)

Примеры:
```
/factchecker wowcasual/diablo_iv/season_8_2026_03_15.md
/factchecker technocrats/ai_agents_2026_03_15.md --deep
/factchecker wowcasual/wow/midnight_announce.md --lang en
```

Аргументы: $ARGUMENTS

## ЭТАП 1: Анализ статьи

### Шаг 1: Прочитай статью

1. Прочитай файл из `docs/{путь}`
2. Извлеки из YAML frontmatter:
   - `sources_used` — список использованных источников (name + url)
   - `title`, `domain`, `format`, `topic` — метаданные статьи
3. Если `sources_used` отсутствует или пуст — отметь это, но продолжай (перекрёстная проверка всё равно возможна)

### Шаг 2: Извлеки проверяемые факты

Прочитай статью и составь список **конкретных проверяемых утверждений**. Для каждого факта запиши:
- **Утверждение** — точная цитата или парафраз из статьи
- **Тип** — дата, число/статистика, цитата, событие, название, механика (для игровых статей), технический факт
- **Критичность** — высокая (ключевой факт статьи), средняя (поддерживающий факт), низкая (деталь/контекст)

Сосредоточься на:
- Даты (релизов, патчей, событий)
- Числа и статистика (цены, показатели, рейтинги)
- Имена (разработчиков, компаний, персонажей — правильность написания)
- Конкретные утверждения о событиях ("компания X объявила Y", "в патче Z добавили W")
- Цитаты (если есть — точность атрибуции)
- Технические/игровые механики (если описаны конкретные числа, проценты, формулы)

**Не проверяй**: мнения автора, субъективные оценки, стилистические обороты, общеизвестные факты.

Покажи пользователю список извлечённых фактов (кратко, таблицей):
```
Извлечено N проверяемых фактов:

| # | Утверждение (кратко) | Тип | Критичность |
|---|---------------------|-----|-------------|
| 1 | Сезон 8 стартует 15 марта | дата | высокая |
| 2 | Новый класс — Спиритборн | название | высокая |
```

## ЭТАП 2: Верификация по оригинальным источникам

### Шаг 3: Загрузи оригинальные источники

Для каждого URL из `sources_used`:
1. Загрузи страницу через WebFetch
2. Извлеки текстовый контент
3. Если страница недоступна — отметь это в отчёте

### Шаг 4: Сверь факты с источниками

Для каждого проверяемого факта:
1. Найди соответствующее место в оригинальном источнике
2. Сравни утверждение в статье с оригиналом
3. Определи статус:
   - **Подтверждён** — факт точно соответствует источнику
   - **Искажён** — факт присутствует в источнике, но с отличиями (укажи какими)
   - **Не найден в источниках** — факт не обнаружен ни в одном из указанных источников
   - **Источник недоступен** — не удалось загрузить источник для проверки

---

## ЭТАП 3: Перекрёстная проверка через веб-поиск

### Шаг 5: Составь поисковые запросы

На основе ключевых фактов статьи составь 3-5 поисковых запросов (или 5-10 при `--deep`):
- Используй ключевые сущности из статьи (названия, даты, имена)
- Формулируй запросы на языке `--lang` (или определи из статьи)
- Для игровых тем: добавляй название игры + ключевой термин
- Для техно-тем: добавляй название технологии/компании + контекст

### Шаг 6: Выполни поиск и собери альтернативные источники

1. Используй WebSearch для каждого запроса
2. Из результатов выбери 3-7 наиболее релевантных страниц (авторитетные издания, официальные сайты)
3. Загрузи выбранные страницы через WebFetch
4. Извлеки факты, которые пересекаются с проверяемыми утверждениями

### Шаг 7: Перекрёстная сверка

Для каждого проверяемого факта:
1. Сравни с информацией из альтернативных источников
2. Определи статус:
   - **Подтверждён** — 2+ независимых источника подтверждают
   - **Частично подтверждён** — найден в 1 источнике или с незначительными расхождениями
   - **Противоречие** — альтернативные источники содержат другую информацию (укажи какую)
   - **Не удалось проверить** — нет релевантных результатов по этому факту

## ЭТАП 4: Итоговый отчёт

### Шаг 8: Сформируй отчёт

Выведи структурированный отчёт:

```
## Отчёт фактчекинга

**Статья**: {title}
**Файл**: docs/{path}
**Дата проверки**: {today}

### Сводка

- Всего проверяемых фактов: N
- Подтверждено: N (✓)
- Искажено/Противоречия: N (✗)
- Не удалось проверить: N (?)

### Оценка достоверности: {ВЫСОКАЯ / СРЕДНЯЯ / НИЗКАЯ / КРИТИЧЕСКАЯ}

{Краткий вывод — 1-2 предложения}

---

### Детальные результаты

| # | Утверждение | Источники | Веб-поиск | Комментарий |
|---|------------|-----------|-----------|-------------|
| 1 | Сезон 8 стартует 15 марта | ✓ Подтверждён | ✓ 3 источника | — |
| 2 | Новый класс Спиритборн | ✗ Искажён | ✗ Противоречие | В источнике: "Spiritborn", в статье: другое описание |

---

### Проблемные факты (если есть)

Для каждого факта со статусом "Искажён" или "Противоречие":
1. **Утверждение в статье**: "..."
2. **Что в источнике**: "..."
3. **Что в альтернативных источниках**: "..."
4. **Рекомендация**: конкретное исправление

---

### Источники проверки

**Оригинальные источники**:
- [name](url) — доступен/недоступен

**Альтернативные источники (веб-поиск)**:
- [title](url) — использован для проверки фактов #1, #3, #5
```

### Шаг 9: Определи оценку достоверности

- **ВЫСОКАЯ** — все факты высокой критичности подтверждены, нет искажений
- **СРЕДНЯЯ** — есть незначительные расхождения или часть фактов не удалось проверить, но ключевые факты верны
- **НИЗКАЯ** — обнаружены искажения в фактах средней/высокой критичности
- **КРИТИЧЕСКАЯ** — обнаружены серьёзные искажения в ключевых фактах статьи

### Шаг 10: Предложи исправления (если нужно)

Если найдены проблемные факты, спроси:
> Найдены расхождения в N фактах. Хочешь, чтобы я исправил их в статье?

**СТОП — жди ответа. Не вноси правки без явного подтверждения.**

Если пользователь подтвердит:
1. Прочитай файл статьи
2. Исправь только те факты, которые были отмечены как "Искажён" или "Противоречие"
3. Не меняй стиль, структуру или авторский тон — только фактические данные
4. Обнови frontmatter: добавь `factchecked: YYYY-MM-DD`
5. Покажи diff изменений

---

## Важные правила

- Не меняй статью без явного подтверждения пользователя
- Различай факты и мнения — мнения не подлежат фактчекингу
- Если оригинальные источники противоречат друг другу — отметь это в отчёте, а не выбирай "правильную" версию
- Если статья не содержит `sources_used` — пропусти этап 2, но обязательно выполни этап 3
- При перекрёстной проверке отдавай приоритет: официальные сайты > крупные издания > блоги > форумы
- Для игровых статей: официальные патчноуты и блог-посты разработчиков — самый авторитетный источник
- Для техно-статей: пресс-релизы компаний, исследовательские статьи, авторитетные издания (Ars Technica, The Verge, etc.)
- Пиши отчёт на **русском** языке (поисковые запросы — на языке статьи или `--lang`)
- Не сохраняй отчёт в файл — выводи только в чат (если пользователь не попросит иначе)
