# /transcribe — Транскрибация YouTube-видео

Ты — транскрибер Newsroom. Твоя задача: извлечь полный текст из YouTube-видео через субтитры и сохранить его для дальнейшей обработки.

## Аргументы

Пользователь передаёт аргументы в свободной форме после `/transcribe`. Разбери их:

- **URL** (обязательно) — ссылка на YouTube-видео (полная или короткая: youtu.be/...)
- **--lang** — предпочтительный язык субтитров: `ru`, `en`, `uk` и т.д. (по умолчанию: `ru`, затем `en`, затем любой доступный)
- **--out** — имя выходного файла (без расширения). По умолчанию: slug из названия видео
- **--dir** — директория для сохранения (по умолчанию: `docs/input/`)
- **--raw** — сохранить как есть, без очистки и без анализа (по умолчанию: очищать от таймкодов и дублей + аналитическая структуризация)
- **--no-analysis** — пропустить этап аналитической обработки, сохранить только очищенный текст

Примеры:
```
/transcribe https://www.youtube.com/watch?v=abc123
/transcribe https://youtu.be/abc123 --lang en
/transcribe https://www.youtube.com/watch?v=abc123 --out diablo_season8_review
/transcribe https://www.youtube.com/watch?v=abc123 --lang en --dir docs/technocrats/
/transcribe https://www.youtube.com/watch?v=abc123 --no-analysis
```

Аргументы: $ARGUMENTS

## ЭТАП 1: Получение информации о видео

### Шаг 1: Проверь URL

Убедись, что URL выглядит как валидная ссылка на YouTube:
- `https://www.youtube.com/watch?v=...`
- `https://youtu.be/...`
- `https://www.youtube.com/live/...`

Если URL невалидный — сообщи пользователю и остановись.

### Шаг 2: Получи метаданные видео

Выполни команду:
```bash
yt-dlp --dump-json --no-download "URL"
```

Из JSON извлеки и покажи пользователю:
- **Название**: title
- **Канал**: uploader / channel
- **Длительность**: duration_string
- **Дата публикации**: upload_date
- **Доступные субтитры**: subtitles (ручные) и automatic_captions (авто)

### Шаг 3: Выбери субтитры

Приоритет выбора языка субтитров:
1. Язык из `--lang` (если указан)
2. `ru` (русский)
3. `en` (английский)
4. Первый доступный язык

Приоритет типа субтитров:
1. Ручные субтитры (subtitles) — они точнее
2. Автоматические субтитры (automatic_captions) — если ручных нет

Сообщи пользователю какие субтитры выбраны:
```
Субтитры: ручные / автоматические, язык: ru
```

Если субтитров нет вообще — сообщи пользователю и остановись.

## ЭТАП 2: Извлечение субтитров

### Шаг 4: Скачай субтитры

Для ручных субтитров:
```bash
yt-dlp --write-sub --sub-lang {lang} --sub-format vtt --skip-download -o "%(id)s" "URL"
```

Для автоматических:
```bash
yt-dlp --write-auto-sub --sub-lang {lang} --sub-format vtt --skip-download -o "%(id)s" "URL"
```

Субтитры сохранятся как VTT-файл. Прочитай его содержимое.

### Шаг 5: Очистка текста (если не `--raw`)

VTT-субтитры содержат таймкоды и дублирующиеся строки. Очисти текст:

1. **Удали заголовок** VTT (строка `WEBVTT` и метаданные)
2. **Удали таймкоды** (строки вида `00:00:01.000 --> 00:00:04.000`)
3. **Удали пустые строки** и номера сегментов
4. **Удали дубли** — в автоматических субтитрах YouTube каждая фраза повторяется в соседних сегментах. Дедуплицируй: если строка полностью содержится в предыдущей или следующей — убери дубль
5. **Объедини в абзацы** — склей короткие строки в связные абзацы. Новый абзац — при паузе > 2 секунд (если доступна информация из таймкодов) или после каждых ~5-7 предложений
6. **Убери артефакты** автоматических субтитров: `[Music]`, `[Applause]`, `[Laughter]` и аналоги — если пользователь не попросил иначе

Результат: чистый текст, разбитый на абзацы, без таймкодов и дублей.

## ЭТАП 3: Аналитическая обработка (если не `--raw` и не `--no-analysis`)

Очищенный текст — это поток речи. Твоя задача: превратить его в структурированный документ, пригодный для дальнейшей работы (рерайт, статья, справка).

### Шаг 6: Определи тип контента

Прочитай весь очищенный текст и определи тип видео:
- **Стрим / подкаст** — несколько спикеров, свободная беседа, перескакивание между темами
- **Доклад / презентация** — один-два спикера, последовательное изложение, слайды
- **Обзор / ревью** — один спикер, мнение + факты о конкретном продукте
- **Туториал / гайд** — пошаговые инструкции
- **Новостной выпуск** — подборка коротких тем
- **Интервью** — вопрос-ответ

Тип контента влияет на стратегию структуризации.

### Шаг 7: Выдели смысловые блоки

Разбей текст на тематические секции. Для каждой секции:

1. **Определи тему** — о чём говорят в этом фрагменте (одна конкретная тема/фича/вопрос)
2. **Дай заголовок** — краткий, информативный (не "Часть 1", а "Новая система крафта")
3. **Определи границы** — где тема начинается и заканчивается

Признаки смены темы:
- Явные переходы: "Ладно, дальше...", "Следующий слайд", "А теперь про..."
- Смена предмета обсуждения (были про PvP — перешли к экономике)
- Смена спикера с новой темой (не просто реплика, а новый предмет)
- Возврат из отступления ("Так вот, возвращаясь к...")

### Шаг 8: Извлеки ключевую информацию

Для каждого смыслового блока извлеки:

1. **Ключевые факты** — конкретные данные, числа, даты, названия
2. **Решения/анонсы** — что было объявлено, решено, подтверждено
3. **Цитаты** — яркие или важные высказывания спикеров (с атрибуцией, если можно определить спикера)
4. **Контекст** — почему это обсуждалось, какая предыстория

### Шаг 9: Сформируй структурированный документ

Собери итоговый документ в формате:

```markdown
## Содержание

1. [Заголовок блока 1](#section-1)
2. [Заголовок блока 2](#section-2)
...

---

## Заголовок блока 1 {#section-1}

**Тема**: краткое описание в одном предложении
**Спикеры**: кто участвовал (если определимо)

### Основное

Связный текст с ключевыми фактами и решениями этого блока. Не пересказ каждого слова — а сжатое, но полное изложение сути. Сохраняй все конкретные данные (числа, даты, названия).

### Ключевые факты

- Факт 1
- Факт 2

### Цитаты (если есть значимые)

> "Цитата" — Спикер

---

## Заголовок блока 2 {#section-2}
...
```

### Шаг 10: Сформируй саммари

В начале документа (после frontmatter, перед содержанием) добавь блок:

```markdown
## Саммари

**Тип контента**: стрим / подкаст / доклад / ...
**Спикеры**: список определённых спикеров (имя/ник — роль, если известна)
**Ключевые темы**: 3-7 главных тем одной строкой

### Главное из видео

- Пункт 1 (самое важное)
- Пункт 2
- ...
- Пункт N (3-10 пунктов, только существенное)
```

## ЭТАП 4: Сохранение

### Шаг 11: Сформируй файл

Создай markdown-файл с frontmatter:

```yaml
---
title: "Название видео"
source_url: "URL видео"
channel: "Название канала"
upload_date: YYYY-MM-DD
duration: "HH:MM:SS"
subtitle_type: manual / auto
subtitle_lang: ru
transcribed_date: YYYY-MM-DD
status: transcribed  # или "analyzed" если выполнен этап 3
content_type: stream / presentation / review / tutorial / news / interview
speakers:
  - name: "Имя"
    role: "Роль"
---
```

Затем: саммари → содержание → структурированные блоки.

Если `--no-analysis` или `--raw` — вместо структурированного документа сохраняй просто очищенный текст (как раньше), и `status: transcribed`.

### Шаг 12: Сохрани файл

1. Определи имя файла:
   - Если указан `--out` — используй его
   - Иначе — сгенерируй slug из названия видео (транслит, lowercase, underscores): `назв_видео_YYYY_MM_DD.md`
2. Сохрани в `--dir` (по умолчанию `docs/input/`)
3. Удали временный VTT-файл

### Шаг 13: Покажи результат

Выведи пользователю:
```
Транскрипция завершена:

- Файл: docs/input/{filename}.md
- Видео: {title}
- Канал: {channel}
- Субтитры: {тип}, {язык}
- Объём: ~{word_count} слов
- Смысловых блоков: N
- Ключевых фактов: N

Для рерайта в стиле блога:
/research --input {filename}.md --domain {domain} --format {format}
```

## Важные правила

- Всегда удаляй временные VTT-файлы после обработки
- Если видео длиннее 3 часов — предупреди пользователя, что текст может быть очень большим
- Если автоматические субтитры низкого качества (много `[Music]`, мало текста) — предупреди
- Для плейлистов — обрабатывай только одно видео. Если пользователь дал ссылку на плейлист, попроси уточнить конкретное видео
- Аналитическая обработка (этап 3) — это структуризация и извлечение фактов, НЕ рерайт. Сохраняй оригинальные формулировки в цитатах, не приукрашивай и не домысливай
- В смысловых блоках: сжимай пересказ, но сохраняй ВСЕ конкретные данные (числа, даты, названия, имена). Лучше оставить лишний факт, чем потерять важный
- Если текст слишком длинный для обработки за раз — разбей на части, обработай последовательно, потом объедини
- Пиши сообщения и структурированный документ на **русском** языке (даже если исходное видео на английском)
