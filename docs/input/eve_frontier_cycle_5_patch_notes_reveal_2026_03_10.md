- Миграция на **Sweet Chain** — запуск в среду с Cycle 5

## Обфускация локаций {#section-3}

**Тема**: скрытие точных координат построек игроков благодаря Sweet Chain
**Спикеры**: CCP Yoten, CCP Good Fellow

### Основное

Одно из первых преимуществ Sweet Chain — обфускация расположения структур (location obfuscation). Ранее все структуры были видны на блокчейне, и игроки могли легко найти кого угодно. Теперь координаты структур видны только как хэш.

В планах — возможность для владельца структуры добровольно раскрыть свои координаты (broadcast). Например, если игрок управляет сетью топливных складов, он может выбрать сделать их видимыми. Эта фича в разработке для point release.

### Ключевые факты

- Структуры теперь видны только как **хэш** — точные координаты скрыты
- **Раскрытие координат (broadcast)** — будущая фича, в разработке для point release
- Решает проблему, когда враги легко находили сетевые ноды и базы

## Система сигнатур и сканирования {#section-4}

**Тема**: новая механика обнаружения объектов в пространстве — пассивная система наблюдения
**Спикеры**: CCP Yoten, CCP Good Fellow, CCP Bowman

### Основное

Вводится **Passive Observation System** — радикально новый подход к обнаружению. Вместо бинарного «видишь/не видишь» появляется градиент понимания. Корабль постоянно пытается разрешить неизвестные сигнатуры вокруг себя.

**Гравитационные сигнатуры** — первый тип. Определяют массу объекта. Лёгкие корабли обнаруживают тяжёлые раньше, чем тяжёлые — лёгкие. Это создаёт тактику лёгких разведчиков и скрытного подхода.

Линия видимости (line of sight) влияет на сканирование: если объект за астероидом — сигнатура пропадает. Информация деградирует со временем, если цель выходит из зоны видимости. Есть dead reckoning — статичные объекты разрешаются быстрее при повторном обнаружении.

В будущем планируются: EM-сигнатуры, тепловые сигнатуры (от горнодобычи), коммуникационные сигнатуры (флоты), активное сканирование (за счёт собственной сигнатуры), ложные сигнатуры (обманки/дроны-декои).

### Ключевые факты

- **Passive Observation System** — корабль автоматически пытается разрешить сигнатуры вокруг
- Первый тип: **гравитационные сигнатуры** (определение массы)
- Маленький корабль обнаруживает большой **раньше**, чем большой — маленький
- **Линия прямой видимости** (line of sight) влияет на сканирование: за препятствием — сигнал пропадает
- Информация **деградирует** со временем при потере контакта
- Звуковой эффект: тикающий звук (как счётчик Гейгера) при появлении новой сигнатуры
- **Не сбалансировано** пока — тюнинг будет проводиться на основе обратной связи
- Доступно для хакатона
- Будущие типы сигнатур: **EM**, **тепловые**, **коммуникационные** (не в этом билде)
- Активное сканирование — прототип есть, в разработке

### Цитаты

> "Everybody's game is about to drastically change as you need to find out what is in the world." — CCP Good Fellow

> "Information is only relevant while it's relevant." — CCP Good Fellow

## Прогрессия персонажей: шеллы, гнёзда и короны {#section-5}

**Тема**: расширение системы шеллов — производство, хранение, улучшение тел персонажей
**Спикеры**: CCP Yoten, CCP Good Fellow

### Основное

Значительное расширение системы прогрессии персонажей из Cycle 4. Введены **shell industry**, **shell fittings** и **crowns**.

Два новых здания: **Nursery** (производство шеллов) и **Nest** (хранение шеллов). Три новых производимых шелла: Aggressive, Reaping и Rugged. Производство одного шелла занимает ~10 часов. Базовый шелл по-прежнему доступен бесплатно.

**Короны (Crowns)** создаются из памятей (memories) через Ascension Protocol и используются для улучшения шеллов. Обновлён character sheet и Ascension Protocol UI. Короны — persistent items, которые **можно потерять**.

### Ключевые факты

- **Nursery** — производство шеллов (тел)
- **Nest** — хранение шеллов
- Три новых шелла: **Aggressive**, **Reaping**, **Rugged**
- Время производства шелла: **~10 часов**
- **Crowns** — создаются из memories, улучшают шеллы, являются persistent items, **можно потерять**
- Character sheet полностью переработан
- Ascension Protocol UI обновлён
- Базовый incarnation shell остаётся бесплатным

## Улучшения камеры {#section-6}

**Тема**: новые режимы камеры и кинематографические улучшения
**Спикеры**: CCP Yoten, CCP Bowman

### Основное

Введены несколько новых камер, над которыми работал арт-интерн CCP Manbjorn:

- **Chase camera (C)** — мягко следует за кораблём сзади. В отличие от старого track, более плавная, кинематографичная
- **Tracking camera (T)** — фокусируется на выбранной цели и корабле одновременно, удерживая оба в кадре
- **Обновлённый телескоп** — значительно более плавный зум, без прежней дёрганости. Работают над чувствительностью при разных уровнях зума + пользовательские настройки
- **Новая камера отстыковки** — более плавная и кинематографичная
- **Composition manager** — автоматически улучшает визуальную композицию кадра

В режиме телескопа виден прицел для будущего **manual targeting/firing mode** — он не включён в Cycle 5, но разрабатывается.

### Ключевые факты

- **Chase camera** — клавиша **C**
- **Tracking camera** — клавиша **T**
- **Manual firing/targeting mode** — в разработке, ожидается "скорее, чем позже"
- Manual targeting = любое взаимодействие на расстоянии, не только стрельба

## Строительство баз: construction sites и турели {#section-7}

**Тема**: обновления строительства — construction sites, ребаланс турелей, множественные якорные точки
**Спикеры**: CCP Yoten, CCP Good Fellow, CCP Bowman

### Основное

**Construction sites** — новый способ строительства. Вместо прямой постройки из инвентаря корабля: ставишь construction site → подвозишь ресурсы из любого хранилища поблизости → строится. Любой игрок может вкладывать ресурсы в чужие construction sites. Владелец — тот, кто поставил site. Время строительства всех структур **снижено на 20%**.

**Полевые структуры** (portable/field structures) вне базы по-прежнему требуют ресурсы в карго.

**Турели** полностью переработаны: три новых типа — mini turret, turret, heavy turret (малые/средние/крупные цели). Оружие теперь встроено в турели (не применяется отдельно), что позволяет балансировать их независимо. Баланс сдвинут в сторону защитников.

**Множественные якорные точки** — от 5 до 20 якорей на L-point. Ограничение: один network node на L-point, но несколько игроков могут строить рядом (аналог tribal base building).

Ограничение на количество турелей снято — теперь зависит только от мощности (power).

Названия структур обновлены с плейсхолдеров. Foam пересчитан: 20 новых foam = 2 старых по стоимости. Всё стоит столько же или дешевле.

### Ключевые факты

- **Construction sites** — новый способ постройки, ресурсы из любого хранилища
- Любой игрок может вкладывать ресурсы в construction site
- Время строительства **снижено на 20%** для всех структур
- **3 новых типа турелей**: mini turret (малые цели), turret (средние), heavy turret (крупные)
- Оружие **встроено** в турели — балансируются отдельно
- Баланс сдвинут **в сторону защитников**
- Лимит турелей убран — зависит **только от power**
- **5-20 якорных точек** на L-point
- **1 network node** на L-point
- Foam пересчитан: **20 новых = 2 старым** по стоимости
- Новая анимация строительства
- Destruction и rebuilding — **не в Cycle 5**, в ближайшем follow-up

## Орбитальные зоны и NPC-оверхол {#section-8}

**Тема**: орбитальные зоны развёрнуты на всю вселенную, полный переработка NPC
**Спикеры**: CCP Yoten, CCP Good Fellow, CCP Bowman, CCP Overload

### Основное

**Орбитальные зоны** (ранее «landscapes») теперь в каждой системе. Все предыдущие данжи и сайты (ресурсные, NPC) **полностью удалены**. Для добычи ресурсов нужно заходить в орбитальные зоны. Добавлено **15 новых версий** зон (к 5 из Cycle 4), все уникальные — с новыми структурами, графикой, волюметрическими облаками.

**NPC-оверхол**: NPC теперь имеют цели, задачи и память. Они перемещаются между точками интереса, включая **базы игроков**. Старые группы фералов (Termitis, Okas, Sakatas, Oza Ter) удалены, введены новые clades, соответствующие лору игры.

NPC **больше не дропают лут** напрямую. Лут теперь из обломков (wrecks), информационных и ресурсных кэшей. NPC охраняют ценные объекты. Некоторые NPC **торгуют** с игроками. В будущем NPC будут salvageable, а особо ценные NPC будут давать особые награды.

Распределение ресурсов во вселенной обновлено. Это системное изменение — можно быстро корректировать без крупных патчей.

### Ключевые факты

- Орбитальные зоны — **в каждой системе**
- Все старые данжи и сайты **удалены**
- **15 новых** уникальных орбитальных зон + 5 из Cycle 4 = **20 вариантов**
- NPC имеют **цели, задачи, память**, перемещаются по точкам интереса
- NPC **приходят на базы игроков**
- Старые фералы (Termitis, Okas, Sakatas, Oza Ter) **удалены**, новые clades по лору
- NPC **не дропают лут** — лут из обломков и кэшей
- Некоторые NPC **торгуют** с игроками
- В будущем: NPC salvageable, ценные NPC = ценные награды
- Обновление распределения ресурсов — **системное**, быстро корректируется
- Вселенная: ~**25 000 солнечных систем**

### Цитаты

> "They have objectives. They have tasks. They have goals. They remember. They think. And they're like crows. They will move between points of interest in a star system, including your base." — CCP Yoten

## Балансировка кораблей {#section-9}

**Тема**: новый корабль Lie и масштабная балансировка всего фригатного ряда
**Спикеры**: CCP Yoten

### Основное

**Новый корабль Lie** — фрегат, микс между MCF и Half. Быстрый, маневренный. Ранее был известен как Microparo в Cycle 1-2. Soul rider Solinaria уже использовал единственный Lie на ивенте Zerotti.

**Tatis** (ранее лучший корабль на все случаи) ослаблен: перераспределены слоты, увеличена скорость, снижена маневренность, изменена угловая скорость, убрана возможность ставить Tier 2 двигатели на десторойеры.

Остальные фрегаты **усилены**: Recurve получил больше power grid и скорости; Reaver — больше маневренности и угловой скорости; Half стал тяжёлым боевым фрегатом; MCF — скорректирована мобильность.

Крупные корабли: **Maul** — перераспределены слоты, добавлен дополнительный high slot для турелей. **Laura** — масса увеличена. **Chumach** — масса уменьшена.

**Adaptive Kinetic Armor Weave** изменён + добавлен **Weave 4** на рынок.

### Ключевые факты

- Новый фрегат **Lie** (микс MCF + Half, быстрый)
- **Tatis** ослаблен: меньше маневренности, нет Tier 2 двигателей на десторойерах
- **Recurve**: +power grid, +скорость, -маневренность, +угловая скорость
- **Reaver**: +маневренность, +угловая скорость
- **Half**: переориентирован на тяжёлый боевой фрегат
- **MCF**: скорректирована мобильность
- **Maul**: новый high slot для турелей
- **Laura**: масса увеличена
- **Chumach**: масса уменьшена
- Масса теперь важна для **обнаружения сигнатурами**
- **Weave 4** добавлен на рынок

## Новые модули, рифты и стартовые регионы {#section-10}

**Тема**: новые двигатели, микро-рифты и обновление начальных зон
**Спикеры**: CCP Yoten

### Основное

Введены три новых модуля передвижения: **Hop**, **Skip** и **Leap** — для быстрого перемещения в орбитальных зонах. Стартовый корабль теперь поставляется с модулем **Skip**. Для более крупных кораблей модули нужно производить.

**Два новых типа рифтов**: микро-рифты — есть практически в каждой системе. Дают небольшое количество crude (не высокого качества), достаточно для базового выживания и перемещения.

Стартовые регионы обновлены для более интересного начала игры.
