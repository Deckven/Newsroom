# Claude Code: агенты, скиллы, хуки и MCP — полный разбор архитектуры

Claude Code — это официальный CLI-инструмент Anthropic для работы с моделями Claude прямо из терминала. Но под капотом это не просто чат-бот в консоли. Это расширяемая платформа с собственной архитектурой плагинов, системой агентов и протоколом интеграции с внешними сервисами. Разберёмся, из чего всё это состоит.

## Инструменты (Tools): базовый слой

Инструменты — это встроенные возможности Claude Code для взаимодействия с окружением. Они делятся на несколько категорий:

- **Файловые операции**: `Read` (чтение файлов), `Write` (создание/перезапись), `Edit` (точечные правки), `NotebookEdit` (Jupyter-ноутбуки)
- **Поиск**: `Glob` (поиск файлов по паттерну), `Grep` (поиск по содержимому через regex), `WebSearch` (веб-поиск), `WebFetch` (загрузка веб-страниц)
- **Терминал**: `Bash` — выполнение shell-команд
- **Агенты и скиллы**: `Agent` (запуск субагентов), `Skill` (вызов скиллов)
- **Взаимодействие с пользователем**: `AskUserQuestion`, `EnterPlanMode`, `ExitPlanMode`

Часть инструментов работает без подтверждения (Read, Glob, Grep — только чтение), другие требуют разрешения пользователя (Edit, Write, Bash — потенциально изменяют файлы или систему).

**Система разрешений** настраивается через `settings.json`:

```json
{
  "permissions": {
    "allow": ["Bash(npm run *)", "Read", "Edit"],
    "deny": ["Bash(rm *)"]
  }
}
```

Приоритет: deny > ask > allow. Первое совпадение побеждает.

## Агенты (Subagents): делегирование задач

Агенты — это специализированные AI-ассистенты, работающие в изолированных контекстных окнах. Каждый агент получает собственный системный промпт, ограниченный набор инструментов и независимую историю диалога.

### Встроенные типы агентов

| Тип | Назначение | Модель |
|-----|-----------|--------|
| **general-purpose** | Универсальный — исследование, код, многошаговые задачи | Наследует от родителя |
| **Explore** | Быстрое исследование кодовой базы (только чтение) | Haiku (быстро и дёшево) |
| **Plan** | Проектирование архитектуры и планов реализации | Наследует от родителя |
| **claude-code-guide** | Ответы на вопросы о Claude Code, API, Agent SDK | Наследует от родителя |

### Как создать своего агента

Агенты определяются в файлах `.claude/agents/<name>/SKILL.md` с YAML-frontmatter:

```yaml
---
name: code-reviewer
description: Expert code review specialist
tools: Read, Grep, Glob
disallowedTools: Write, Edit
model: sonnet
maxTurns: 20
---

Ты — ревьюер кода. Анализируй код, находи баги и проблемы.
```

Ключевые параметры:
- **tools** / **disallowedTools** — белый и чёрный списки инструментов
- **model** — переопределение модели (sonnet, opus, haiku)
- **maxTurns** — лимит ходов агента
- **isolation: worktree** — запуск в изолированной копии репозитория (отдельный git worktree)
- **memory** — персистентная память агента между сессиями

### Foreground vs. Background

- **Foreground** (по умолчанию): основной диалог ждёт завершения агента. Запросы разрешений проходят к пользователю.
- **Background**: агент работает параллельно. Все разрешения выдаются заранее. Пользователь продолжает работу и получает уведомление по завершении.

### Ограничения

Субагенты **не могут** создавать других субагентов (нет вложенности). Они не наследуют скиллы родительской сессии — нужные скиллы перечисляются явно в поле `skills`.

## Скиллы (Skills): переиспользуемые промпты

Скиллы — это переиспользуемые инструкции, которые расширяют знания Claude или дают пошаговый алгоритм для конкретных задач. Вызываются через `/skill-name`.

### Где живут скиллы

| Расположение | Область | Расшариваемость |
|-------------|---------|-----------------|
| `.claude/skills/<name>/SKILL.md` | Проект | Да (git) |
| `~/.claude/skills/<name>/SKILL.md` | Персональный | Нет |
| Плагины | Область плагина | Через плагин |

### Формат файла

```markdown
---
name: deploy
description: Deploy application to staging or production
argument-hint: [environment]
disable-model-invocation: true
allowed-tools: Bash, Read
---

## Деплой приложения

1. Проверь текущую ветку: !`git branch --show-current`
2. Запусти тесты...
3. Задеплой в $ARGUMENTS[0]...
```

### Ключевые возможности

- **Аргументы**: `$ARGUMENTS` — все аргументы; `$1`, `$2` — позиционные
- **Динамический контекст**: `` !`команда` `` выполняется **до** того, как Claude увидит промпт. Результат подставляется на место
- **Контроль вызова**:
  - `disable-model-invocation: true` — только пользователь вызывает вручную (для опасных операций вроде деплоя)
  - `user-invocable: false` — только Claude вызывает автоматически (фоновые знания)
- **Контекст**: `context: fork` — скилл выполняется в изолированном субагенте, не загрязняя основной контекст

### Скиллы vs. Команды

В Claude Code существуют два механизма:
- **Commands** (`.claude/commands/`) — устаревший формат, один файл
- **Skills** (`.claude/skills/`) — современный формат, директория с поддерживающими файлами

Оба создают `/slash-команду`. Skills — предпочтительный путь для новых проектов.

## Хуки (Hooks): детерминированная автоматизация

Хуки — это shell-команды (или HTTP/промпт-решения), которые выполняются при определённых событиях жизненного цикла Claude Code. В отличие от скиллов, хуки **детерминированны** — срабатывают всегда при наступлении события.

### Основные события

| Событие | Когда срабатывает |
|---------|-------------------|
| `PreToolUse` | Перед выполнением инструмента (может заблокировать) |
| `PostToolUse` | После успешного выполнения инструмента |
| `UserPromptSubmit` | При отправке промпта пользователем |
| `SessionStart` | При начале/возобновлении сессии |
| `Stop` | Когда Claude завершает ответ |
| `SubagentStart` / `SubagentStop` | Жизненный цикл субагентов |
| `PreCompact` / `PostCompact` | Компактификация контекста |

### Типы хуков

1. **command** — shell-команда. Получает JSON на stdin, управляет через exit code (0 = ОК, 2 = блокировка)
2. **http** — HTTP POST на внешний endpoint. Для облачных сервисов аудита
3. **prompt** — Claude (Haiku) принимает решение да/нет. Для сложной логики
4. **agent** — полноценный субагент с инструментами проверяет условия

### Пример: защита файлов от редактирования

```json
{
  "hooks": {
    "PreToolUse": [{
      "matcher": "Edit|Write",
      "hooks": [{
        "type": "command",
        "command": "./.claude/hooks/protect-files.sh"
      }]
    }]
  }
}
```

Скрипт проверяет путь файла — если это `.env` или `.git/`, возвращает exit 2 и блокирует операцию.

### Типичные сценарии

- Автоформат кода (Prettier/Black) после каждого редактирования
- Блокировка правок в защищённых файлах
- Desktop-уведомления когда Claude ждёт ввода
- Аудит-лог всех изменений конфигурации
- Переинъекция контекста после компактификации

## MCP (Model Context Protocol): внешние интеграции

MCP — открытый протокол для подключения AI-инструментов к внешним системам. MCP-серверы предоставляют инструменты и ресурсы, которые Claude Code использует наравне со встроенными.

### Архитектура

- **MCP-сервер** — внешний процесс, предоставляющий инструменты (GitHub, Slack, БД, браузер)
- **MCP-клиент** — Claude Code подключается к серверу и использует его инструменты
- **Транспорт**: stdio (подпроцесс), http, sse (Server-Sent Events), ws (WebSocket)

### Конфигурация (.mcp.json)

```json
{
  "mcpServers": {
    "github": {
      "type": "stdio",
      "command": "node",
      "args": ["path/to/github-mcp-server/index.js"],
      "env": { "GITHUB_TOKEN": "${GITHUB_TOKEN}" }
    }
  }
}
```

MCP-инструменты именуются по шаблону `mcp__<сервер>__<инструмент>` — например, `mcp__github__search_repositories` или `mcp__slack__send_message`.

### Расположение конфигурации

- `.mcp.json` в корне проекта — для команды (git)
- `~/.claude/.mcp.json` — персональные серверы
- `mcpServers` в `settings.json` — альтернативный путь
- Frontmatter агентов — привязка серверов к конкретному агенту

## Настройки и конфигурация: иерархия

Claude Code использует многоуровневую систему конфигурации:

| Файл | Область | Расшариваемость |
|------|---------|-----------------|
| `~/.claude/settings.json` | Персональный | Нет |
| `.claude/settings.json` | Проект | Да (git) |
| `.claude/settings.local.json` | Проект (локальный) | Нет (gitignore) |
| `CLAUDE.md` | Инструкции для Claude | Да (git) |
| `.claude/rules/*.md` | Правила по путям | Да (git) |
| `.mcp.json` | MCP-серверы | Опционально |

**Приоритет** (от высшего к низшему): managed settings (энтерпрайз) → CLI-флаги → local settings → project settings → user settings.

### Структура директории .claude/

```
.claude/
├── settings.json          # Настройки проекта
├── settings.local.json    # Локальные переопределения
├── agents/                # Определения агентов
├── skills/                # Определения скиллов
├── commands/              # Команды (legacy)
├── hooks/                 # Скрипты хуков
├── rules/                 # Правила по путям
└── agent-memory/          # Персистентная память агентов
```

## Agent SDK: программные агенты

Claude Agent SDK — это отдельный фреймворк Anthropic для создания автономных AI-агентов на Python или TypeScript, работающих **вне** CLI Claude Code.

| Аспект | Claude Code CLI | Agent SDK |
|--------|----------------|-----------|
| Среда | Интерактивный терминал | Программный код |
| Интерфейс | Промпт-цикл в CLI | Python/TypeScript классы |
| Деплой | Рабочая станция / CI/CD | Где угодно |
| Инструменты | Встроенные + MCP | Кастомные через код |
| Кейс | Интерактивная разработка | Продакшн-агенты |

Agent SDK следует использовать, когда нужен программный контроль: встраивание агента в приложение, продакшн-автоматизация, сложные мультиагентные системы.

## Итого: что есть что

| Механизм | Суть | Когда использовать |
|----------|------|-------------------|
| **Tools** | Встроенные возможности (Read, Edit, Bash...) | Базовые операции |
| **Agents** | Изолированные AI-ассистенты с ограниченным контекстом | Делегирование специализированных задач |
| **Skills** | Переиспользуемые промпты-инструкции | Стандартизированные рабочие процессы |
| **Hooks** | Детерминированные обработчики событий | Автоматизация, валидация, защита |
| **MCP** | Протокол подключения внешних сервисов | Интеграция с GitHub, Slack, БД, браузером |
| **Commands** | Устаревшая форма скиллов | Миграция в skills |
| **Agent SDK** | Фреймворк для программных агентов | Продакшн-агенты вне CLI |

Ключевой архитектурный принцип: **композируемость**. Скиллы могут запускать агентов, агенты могут использовать скиллы, хуки валидируют и тех и других, MCP расширяет доступные инструменты для всех. Всё это настраивается через единую иерархию конфигурации с чёткими правилами приоритетов и областей видимости.
