# Базы на L-Points: кооп-строительство, турели

Строительство баз — одна из ключевых механик EVE Frontier. В Cycle 5 система получила масштабное обновление: кооперативное строительство на L-Points, Construction Sites для совместной доставки материалов и три типа специализированных турелей для обороны. Разберём процесс от выбора точки до полноценной укреплённой базы.

## L-Points: что это и почему они важны

L-Points — это специальные точки в пространстве, пригодные для размещения инфраструктуры. Не каждая координата в космосе подходит для строительства — L-Points определяют, где именно можно закрепить структуры.

### Ключевое нововведение Cycle 5

Несколько игроков теперь могут размещать свои структуры на одном L-Point. Если раньше на одной точке Лагранжа располагалась лишь одна точка для основания базы, то теперь на каждой такой локации добавлено от **5 до 20 новых анкерных точек** (anchor points). При этом действует строгое правило: каждый отдельный игрок может разместить **только один Network Node** в пределах одного L-Point. Это создаёт аналог tribal base building — несколько игроков строят рядом. Разработчики в шутку называют это эффектом «привет, сосед» (hello neighbor) — ситуацией, когда игроки могут буквально жить рядом и «заходить друг к другу одолжить сахара».

На практике это означает:
- Tribe может построить полноценную базу, где каждый участник вносит свои структуры
- Соседство на L-Point требует либо доверия, либо обороноспособности
- Стратегически важные L-Points становятся объектами борьбы
- Заселённые L-Points могут превращаться в настоящие укреплённые поселения — оборонительные кластеры игроков

**Важно:** destruction и rebuilding структур **не вошли в Cycle 5** — CCP планирует добавить их в ближайшем follow-up патче, не дожидаясь следующего крупного milestone.

## Шаг 1: Выбор L-Point

### Критерии выбора

1. **Безопасность** — насколько активна система, есть ли враждебные Tribes поблизости
2. **Ресурсы** — близость к Orbital Zones и рифтам для снабжения
3. **Логистика** — доступность для союзников, расстояние до торговых маршрутов
4. **Стратегическое положение** — контроль над ключевыми переходами
5. **Соседство** — кто уже построил на этом или соседних L-Points

### Рекомендации

- Для первой базы выбирайте L-Point в относительно безопасной системе
- Убедитесь, что поблизости есть Orbital Zones для ресурсов
- Проверьте, нет ли враждебных структур на том же L-Point
- Обсудите выбор с Tribe — база строится совместно

## Шаг 2: Network Node и Construction Site

### Network Node

Network Node — это основа любой базы. Это первая структура, которую нужно разместить. Она определяет принадлежность базы и открывает доступ к дальнейшему строительству.

### Construction Site — новый способ строительства

Construction Sites — ключевое нововведение Cycle 5 в строительстве. Ранее объекты возводились напрямую из меню строительства, но теперь процесс стал более глубоким и интерактивным:

1. **Ставите Construction Site** в нужной точке — сама по себе площадка ещё не является строящимся зданием, это маркер в пространстве, показывающий, что сюда необходимо внести ресурсы
2. **Подвозите ресурсы** из любого хранилища поблизости (не только из карго корабля!) — при нахождении на своей базе вы можете вносить материалы напрямую из любого близлежащего контейнера-хранилища (storage container)
3. **Любой игрок** может вкладывать ресурсы в чужие Construction Sites — можно разделить стоимость постройки с друзьями, загрузив ресурсы пополам; владельцем остаётся тот, кто поставил site
4. Когда все материалы собраны — запускается процесс постройки с **новыми визуальными анимациями строительства** и обновлённой сеткой строительства (building grid)

Для размещения структур используйте **режим строительства (Build Mode)** — нажмите клавишу **B**, затем выберите нужную структуру на вкладке **Core** в меню выбора. Инструменты размещения (placement tools) позволяют расположить объект рядом с собой.

**Ghost Structures (Призрачные структуры):** Construction Sites используют механику «призрачных структур». Вы можете расставить контуры (голограммы) всех желаемых зданий заранее, спланировав идеальную компоновку базы, **не тратя ресурсов**. После размещения макета кликните по нему правой кнопкой мыши и начинайте подвозить материалы по мере добычи. Это идеальный инструмент для совместной игры — любой союзник из Tribe может подлететь к вашим Ghost Structures и внести свои ресурсы.

**Полевые структуры** (portable/field structures), размещаемые вне базы, по-прежнему требуют ресурсы непосредственно в карго корабля. Пример полевой структуры — **Field Refinery** (полевой перерабатывающий завод), который разворачивается примерно за **~30 секунд** и может быть позже демонтирован обратно в трюм. Совет: стройте мини-мастерские (принтер + завод + хранилище) прямо в зонах добычи ресурсов — это экономит время при малом трюме начальных кораблей.

### Сниженное время строительства

Время строительства **всех структур снижено на 20%** в Cycle 5. Названия структур обновлены с плейсхолдеров на финальные. Foam (строительный материал) пересчитан: **20 новых единиц foam = 2 старым** по стоимости. Всё стоит столько же или дешевле, чем раньше.

### Преимущества кооперативного строительства

- Ресурсы из любого хранилища, не только из карго — логистика упрощается
- Любой может помочь — не нужна авторизация для вклада материалов
- Распределение нагрузки: никто не должен добывать все ресурсы в одиночку
- Специализация: каждый добывает то, что умеет лучше
- Сплочение Tribe: совместная работа укрепляет связи

## Шаг 3: Размещение Smart Assemblies

После создания Network Node размещайте ключевые структуры:

### Приоритет 1: Производство и хранение

- **Smart Storage Unit (SSU)** — хранилище для ресурсов и предметов
- **Nursery** — производство Shell (критически важно)
- **Nest** — хранилище для запасных Shell

### Приоритет 2: Инфраструктура

- **Smart Gate** — врата для быстрого перемещения союзников
- **Дополнительные SSU** — торговые терминалы, квестовые системы

### Приоритет 3: Оборона

- **Smart Turrets** — автоматизированная защита (подробнее ниже)
- **Стены** — барьеры для замедления атакующих

## Шаг 4: Оборонительная система — три типа турелей

### Установка турелей

Smart Turret размещается в радиусе 25 км от другой Smart Assembly и автоматически прикрепляется к ней.

**Что было раньше:** в предыдущих циклах существовало жёсткое ограничение — максимум **6 турелей** на локацию, при этом игрокам приходилось **вручную устанавливать обычное корабельное оружие** на турели. Это делало систему крайне сложной для балансировки и смещало баланс в пользу атакующих.

**Что изменилось в Cycle 5:** Разработчики убрали старую механику и заменили её тремя новыми готовыми типами турелей со **встроенным оружием** (embedded weapons). Ручная установка вооружения больше не требуется — это позволяет CCP балансировать урон и характеристики баз **отдельно от корабельного вооружения**. Лимит в 6 турелей **полностью убран** — теперь ограничение зависит **только от power** (мощности энергосети базы). Вы можете построить столько защитных орудий, сколько способна запитать ваша энергетическая сеть. Каждый тип турели имеет свои собственные требования к потреблению энергии.

**Смена парадигмы:** разработчики прямо признают, что предыдущая система обороны работала плохо и сильно смещала баланс в пользу атакующих. Новая система — целенаправленный шаг, чтобы **передать преимущество в руки защитников** (defenders) и заставить рейдеров прикладывать гораздо больше усилий для успешного нападения.

**Концепция «doom base»:** при достаточном запасе энергии можно окружить свой Network Node **сплошным кольцом из турелей**, создавая настоящую «базу смерти» для любых непрошеных гостей. В сочетании с множественными anchor points на L-Point, группы игроков могут создавать невероятно плотные оборонительные кластеры.

### Autocannon Turret

| Параметр | Значение |
|----------|---------|
| Назначение | Противодействие малым целям |
| Оптимальные цели | Фрегаты, лёгкие корабли |
| Стоимость | Низкая |

Autocannon Turret — первая линия обороны. Эффективна против лёгких кораблей вроде Lie и других фрегатов, пытающихся прорваться к базе. Дешёвая, можно ставить в большом количестве.

### Turret (стандартная)

| Параметр | Значение |
|----------|---------|
| Назначение | Противодействие средним целям |
| Оптимальные цели | Десторойеры, средние корабли |
| Стоимость | Средняя |

Стандартная турель — универсальный выбор. Баланс между уроном и скоростью наведения делает её надёжным решением против наиболее распространённых типов кораблей.

### Railgun Turret

| Параметр | Значение |
|----------|---------|
| Назначение | Противодействие крупным целям |
| Оптимальные цели | Крупные боевые корабли |
| Стоимость | Высокая |

Railgun Turret — тяжёлое вооружение для отражения серьёзных атак. Разрушительный урон по крупным целям, но менее эффективна против быстрых фрегатов.

### Рекомендуемая конфигурация обороны

Для сбалансированной защиты базы (лимит зависит от power):
- Несколько Autocannon Turret — по периметру, против быстрых рейдеров
- 1-2 стандартных Turret — в средней зоне, универсальное покрытие
- 1 Railgun Turret — в центре, против тяжёлых атакующих

## Шаг 5: Программирование логики турелей

Smart Turrets — это не просто «стрелялки». Их поведение определяется смарт-контрактом, который можно настроить.

### Базовые режимы

1. **Агрессивный** — стреляет по всем, кроме белого списка
2. **Оборонительный** — стреляет только по тем, кто атаковал первым
3. **Территориальный** — предупреждение, затем атака при дальнейшем приближении

### Продвинутая логика

- Приоритизация целей по классу корабля
- Координация между несколькими турелями (фокус-огонь)
- Экономическая логика: предложение «откупиться» перед атакой
- Интеграция с Smart Gate: блокировка враждебных кораблей на входе

### Блюпринты

Если вы не программист — используйте готовые блюпринты из сообщества. Множество конфигураций турелей доступно бесплатно. Установка блюпринта — вопрос нескольких кликов.

## Шаг 6: Стены и периметр

Стены — это барьеры, замедляющие атакующих и определяющие периметр базы. Они не неразрушимы, но дают время турелям для поражения целей.

Рекомендации:
- Размещайте стены так, чтобы создать «канал» — зону, через которую атакующим придётся лететь под огнём турелей
- Оставляйте один вход, контролируемый Smart Gate
- Не забывайте о резервном выходе для эвакуации

## Чек-лист строительства базы

1. [ ] Выбрать L-Point с учётом безопасности, ресурсов и логистики
2. [ ] Разместить Network Node
3. [ ] Создать Construction Sites для ключевых структур
4. [ ] Скоординировать доставку материалов с Tribe
5. [ ] Установить SSU для хранения ресурсов
6. [ ] Установить Nursery и Nest
7. [ ] Разместить Smart Gate для быстрого доступа
8. [ ] Установить турели: Autocannon Turret по периметру, стандартные в средней зоне, Heavy в центре
9. [ ] Настроить логику турелей (или загрузить блюпринт)
10. [ ] Установить стены и определить периметр
11. [ ] Провести тестовую атаку для проверки обороны

## Stargates: связь между системами

**Звёздные врата (Stargates)** — самые масштабные структуры в игре. По умолчанию **20 000 систем** (120 000 к релизу) **не соединены** между собой вратами. Игроки перемещаются с помощью прыжковых двигателей (jump drive), ограниченных дальностью. Племена могут самостоятельно строить Stargates, создавая постоянные маршруты между системами и формируя альтернативную карту вселенной.

## Стратегия заселения новой системы

При прыжке в новую систему:
1. **Заранее загрузите в трюм минимальный запас** — по 50-100 единиц feldspar, swag и comet
2. По прибытии **немедленно постройте Refuge** — это ваша точка возрождения при гибели
3. Разверните Field Refinery, Printer и Storage рядом с Refuge
4. Ищите систему с **внутренними и внешними орбиталями** — это гарантирует доступ ко всем базовым ресурсам и топливу

Строительство базы — не разовое действие, а непрерывный процесс. Расширяйте, улучшайте, адаптируйте. Frontier вознаграждает подготовленных.
