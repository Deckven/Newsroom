### 3. Модуль Publisher — мультиплатформенная дистрибуция
- Создан модуль `newsroom/publisher/` с интеграцией 7 платформ
- Контент проверяется человеком перед публикацией (отдельный шаг, не автопубликация)
- WordPress: публикация как draft (черновик)

#### Платформы
| Платформа | Файл | Метод |
|-----------|------|-------|
| WordPress | wordpress.py | REST API, Application Password |
| Telegram | telegram.py | Bot API, auto-split длинных сообщений |
| Discord | discord.py | Webhook, поддержка embeds |
| Twitter/X | twitter.py | API v2, OAuth 1.0a, треды |
| LinkedIn | linkedin.py | API v2, UGC Posts |
| Reddit | reddit.py | OAuth2, self-posts |
| Facebook | facebook.py | Graph API v21.0, Page posts |

#### Архитектура
- `BasePublisher` — единый интерфейс `publish(content, **kwargs) -> PublishResult`
- `PublishResult` — dataclass с полями: platform, success, url, post_id, error, metadata
- `config.py` — все креды из env vars (`.env` файл, не в репо)
- `__init__.py` — фасад: `publish()`, `publish_many()`, `get_publisher()`, `available_platforms()`
- `.env.example` — шаблон переменных окружения
- Credentials: будут предоставлены позже
