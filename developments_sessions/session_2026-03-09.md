## Архитектурные решения

- **Lazy imports** для `anthropic`, `batch.py` — рерайтер опциональный, не тянет тяжёлые зависимости при обычном использовании Newsroom
- **`logging` вместо `rich`** — Newsroom использует стандартный logging, а не rich console
- **`RewriterConfig` вместо `Settings`** — Newsroom передаёт конфиг через dict из YAML, а не через Pydantic Settings / .env

## Новые команды

```bash
# Оценка стоимости анализа
python -m newsroom rewriter-setup analyze --cost-estimate

# Анализ через Batch API (50% дешевле)
python -m newsroom rewriter-setup analyze --use-batch

# Продолжить прерванный анализ
python -m newsroom rewriter-setup analyze --resume

# Комбинация
python -m newsroom rewriter-setup analyze --use-batch --resume
```

## Различия Rewriter (standalone) vs Newsroom (integrated)

| Аспект | Standalone | Newsroom |
|--------|-----------|----------|
| Конфиг | Pydantic Settings + .env | RewriterConfig (dict из YAML) |
| Логирование | rich.Console | logging |
| CLI | Click (4 команды) | argparse (subcommand `rewriter-setup`) |
| Batch API | Встроен в StyleExtractor | Lazy import при `--use-batch` |
| Импорты | `rewriter.*` | `newsroom.rewriter.*` |
| Skill `/rewrite` | В `.claude/skills/` | Зарегистрирован как Claude Code skill |
